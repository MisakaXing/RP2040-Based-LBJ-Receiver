# 1.1 硬件

您可以在/PCB文件夹中找到PCB Gerber，EDA工程文件，BOM表，原理图，请自行焊接组装

需要注意的是某些国产pico开发板的USB-C接口没有下拉电阻，这会导致无法使用PD充电器，请购买前向卖家确认好是否有下拉电阻，若不使用PD充电，可忽略本条

推荐的电池容量是3.7v 1200-1600mah 如果您希望使用更大的电池，可以修改R4/Rprog实现修改充电电流，默认充电电流是500ma，Rprog和充电电流ICHG的换算关系是ICHG=1000/Rprog
PCB上电池接口为XH2.54，电池需要购买XH2.54**反向口**

PCB上不带锂电池保护电路，购买的锂电池必须**自带保护板**！

## 2.1软件实现原理
### 基本原理

如果您对原理不感兴趣，请直接跳转到3.1刷入程序

中国铁路列车会在821.2375Mhz上以3w功率发送POCSAG1200数据，数据类型均为numeric，列车会分两次连续发送两条数据，范例如下：
123 23 123 #Basic msg
列车号 速度 公里标 其中公里标格式最后一位是小数点，即12.3km
204U2390093130U-(2 9U- (-(202011720927939053465000 #extend msg
解码规则如下

| 字段值 | 含义 |
|------|------|
| 204U | 两位ASCII标准码表示列车等级，在这里4U=K，即K字头 |
|2390|机车类型标识号，可以从/main/rp2040-main-program/locos.json找到|
| 0093|机车号|
|30|端位，30表示无端，31为A端，32为B端|
|U-(2 9U- (-(2020|四位的GBK编码汉字，表示当前线路|
|117209279|经度|
|39053465|纬度|
|000|未知|

所有信息都只会出现在对应的位置，位号固定

将两条信息组合即成列车完整信息

需要注意的是并非所有线路均有扩展信息的发送，部分线路只有基础信息，甚至不少动车组列车连基础信息都不发
### 本项目实现

通过操控SX1276的寄存器，调整其为FSK Direct模式，DIO1和DIO2分别输出1200hz时钟和采集到的bit，利用树莓派的PIO在时钟的上升沿采样DIO2的数字输出，寻找同步码并解码POCSAG，进行2bit BCH纠错后依上述解码方式解码，格式化成json返回给主函数，以上代码实现请见lbj_receiver.py


## 3.1刷入及更新工具

当完成PCB硬件焊接时，请从**Release**中下载好预打包的刷入/更新工具Pico-Updater.exe
打开后点击刷新，会自动选中pico，之后点击检查更新即可，系统会自动从Github下载最新版本刷入，同时当有新版本发出时，你也可通过此程序一键更新到最新版本

*如果要使用自动更新功能，请确保不要更改Main.py的版本变量*
```python
Program_ver = 2.1
```
## 4.1日志查看工具

你可以从**Release**中下载Log-viewer.exe来实现日志的查看和导出，机器内置2000条车次存储空间，同时可外接SD卡扩展，SD卡必须格式化成FAT32格式才能被识别

SD卡热插拔功能因影响屏幕刷新已被取消，在开机状态下插入SD卡需要在设置中手动Mount-SD

下载后链接pico点击刷新，从pico提取历史数据，可提取机内保存的历史数据

也可以点击导入JSON历史文件从SD卡中导入JSONL文件实现查看
